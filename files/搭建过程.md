## go-zero新建项目

```shell
# 创建项目。需要下划线，不支持中划线
goctl api new realms_web3_go

cd .\realms_web3_go\
# 管理依赖
go mod tidy
```

go.mod也需要提交到仓库中

✅ 需要提交的文件

1. 核心代码文件
   1. mtnode.go - 主入口文件
   2. internal/ 目录下的所有文件：
      1. nternal/config/config.go - 配置结构
      2. internal/handler/*.go - 路由和处理器
      3. internal/logic/*.go - 业务逻辑
      4. internal/svc/*.go - 服务上下文
      5. internal/types/*.go - 类型定义
2. 配置文件
   1. go.mod - Go 模块依赖声明（必须提交）
   2. go.sum - 依赖校验文件（必须提交）
   3. etc/mtnode-api.yaml - 应用配置文件
   4. mt_node.api - API 定义文件
3. 项目文档
   1. README.md - 项目说明
   2. LICENSE - 许可证文件
4. 其他文件

## 操作数据库的CRDU的代码

运行命令 goctl model mysql ddl -c -src user.sql -dir ., 会生成操作数据库的 CRDU 的代码。

此时的 model 目录：

├── user.sql # 手写
├── usermodel.go # 自动生成
└── vars.go # 自动生成
model 生成的代码注意点

model 这块代码使用的是拼接 SQL 语句，可能会存在 SQL 注入的风险。
生成 CRUD 的代码比较初级，需要我们手动编辑 usermodel.go 文件，自己拼接业务需要的 SQL。参见 usermdel.go 中的 FindByName 方法。

```shell
goctl model pg datasource \
--url "postgres://postgres:postgres@localhost:5432/mt_node?sslmode=disable" \
--schema "mt_auth" \
--table "chain_info" \
--dir . \
-c
```
